1. OVERVIEW
================================================================================

The Material Icons OS uses a client-side theme system that loads themes 
dynamically via the ThemeManager JavaScript object. Each theme consists of:

- CSS file: CSS variable overrides + .system-* classes + background + canvas
- JavaScript file (optional): Canvas animations with mandatory cleanup function
- Assets: Images, fonts, and other resources (optional)

ARCHITECTURE CHANGE (v2.6+):
- OS Shell (.icon, .window, .taskbar, .start-menu) = desktop.html system CSS
- Theme CSS = CSS Variables + .system-* classes + Canvas + Background only
- Themes NO LONGER style OS shell elements directly

ARCHITECTURE CHANGE (v2.7+):
- Icon styling now uses CSS variables (--system-icon-bg, --system-icon-border)
- ThemeManager bugs fixed (canvas reference, CSS loading, background apply)
- Background image handling improved for initial load
- NEW: Desktop Shell System Classes (.system-desktop-*) for enhanced theming
- NEW: Long-press interaction model with ContextMenu API for desktop icons

KEY FEATURES:
- Dynamic Loading: Themes apply without page reload
- Canvas Support: Animated backgrounds with automatic cleanup
- CSS Variables: Easy color theming with custom properties
- Complete Override: Theme CSS variables override all base styles
- Theme Persistence: Selected theme saved to localStorage
- URL Parameters: Apply themes via ?theme=theme-id
- Resource Cleanup: Automatic cleanup of animations, timers, listeners
- Component Library: 90+ .system-* classes for consistent UI theming
- Desktop Classes: 40+ .system-desktop-* classes for OS shell theming [NEW]
- App Container: .system-app for zero-config HTML element styling
- Iframe Sync: Apps in iframes automatically sync with OS theme changes
- Separation of Concerns: OS shell styled in desktop.html, themes override variables
- Icon Theming: Desktop icons fully themeable via CSS variables
- Long-Press System: Touch-first context menus with setupLongPress() utility [NEW]
- Context Menu API: Unified menu system with keyboard/touch support [NEW]

2. CRITICAL RULES (READ FIRST!)
================================================================================

*** THESE RULES ARE NON-NEGOTIABLE ***

1. Theme CSS uses CSS variables to override base styles (NO direct shell styling)
2. Scripts MUST set window.ThemeCleanup = function cleanup() {...} at global scope
3. Canvas MUST stay transparent - use clearRect(), never fillRect() with solid color
4. Event listeners MUST be removed in cleanup using stored references
5. Animation frames MUST be cancelled with cancelAnimationFrame()
6. Use .system-* classes or .system-app container for UI components in apps
7. Iframe apps MUST include os-theme-bridge.js for theme sync
8. DO NOT style OS shell elements (.icon, .window, .taskbar, etc.) in theme CSS
9. Theme CSS should ONLY override :root variables and .system-* classes

*** NEW CRITICAL RULES (v2.7+) ***

10. Theme CSS MUST include icon background/border variables
    - --system-icon-bg, --system-icon-border, --system-icon-bg-hover, --system-icon-border-hover
    - Icons will have no background/border without these overrides

11. Theme CSS body background MUST use full shorthand with !important
    - Use: background: url('bg.png') no-repeat center center fixed !important;
    - NOT: background-color: url('bg.png'); (won't work)
    - Include: min-height: 100dvh !important; min-width: 100vw !important;

12. Theme script canvas references MUST be properly scoped
    - Use local `const canvas` variable in script.js
    - In desktop.html ThemeManager.cleanup(), use `this.canvas` not `canvas`
    - Never reference undefined `canvas` in cleanup functions

13. Desktop Shell Classes (.system-desktop-*) are OPTIONAL enhancement hooks
    - Original class names (.taskbar, .window, .icon) remain unchanged
    - .system-desktop-* classes provide semantic theming hooks
    - Themes can override .system-desktop-* for advanced effects
    - Backward compatible - existing themes work without changes

14. Long-Press Interactions MUST include keyboard fallback
    - Desktop icons must have tabindex="0" and role="button"
    - onkeydown handler for Enter/Space must trigger same action as onShortPress
    - Context menus should be dismissible via Escape key

PRO TIP: Test cleanup early
Apply your theme, then immediately switch to another theme. If animations 
continue or styles persist, your cleanup is broken. Fix it before adding features.

3. FILE STRUCTURE
================================================================================

RECOMMENDED ORGANIZATION:

/webos/
  |-- desktop.html          [Main OS file with ThemeManager + OS Shell CSS]
  |-- themes.json           [Theme registry - REQUIRED]
  |-- theme-doc.txt         [This documentation file]
  |-- os-theme-bridge.js    [SDK for iframe apps - REQUIRED for apps]
  +-- themes/
      +-- your-theme/
          |-- styles.css          [CSS variables + .system-* + .system-desktop-*]
          |-- script.js           [with window.ThemeCleanup]
          |-- preview.jpg         [optional thumbnail for Settings UI]
          |-- background.png      [optional background image]
          +-- assets/
              +-- fonts/

REQUIRED FILES:

File                              | Required | Purpose                     | Key Requirement
----------------------------------|----------|-----------------------------|------------------------
themes.json                       | Yes      | Theme registry/metadata     | Valid JSON, correct paths
themes/your-theme/styles.css      | Yes      | Visual styling              | CSS variables + .system-* only
themes/your-theme/script.js       | Optional | Canvas animations           | Must set window.ThemeCleanup
themes/your-theme/preview.jpg     | Optional | Theme thumbnail in Settings | Recommended: 300x200px
os-theme-bridge.js                | Yes*     | Iframe app theme sync       | *Required if using iframe apps
desktop.html                      | Yes      | OS Shell + System CSS       | Contains all shell styling

Font Assets (Optional but Recommended):
-------------------------------------------------------------------------------
/system/fonts/MaterialIcons-Regular.ttf   | Recommended | Local icon font    | Must match @font-face src
/system/fonts/MaterialIcons-Regular.woff2 | Optional    | Compressed variant | Smaller size, same quality
-------------------------------------------------------------------------------

NOTE: The @font-face declaration in desktop.html MUST reference the exact 
      path and filename of your font file. Mismatched paths will cause icons 
      to render as plain text instead of glyphs.

NOTE: File names can be customized but must match the paths specified in 
      themes.json exactly. Paths are relative to desktop.html.

THEME CSS SHOULD NOT CONTAIN:
X .icon { ... }
X .window { ... }
X .taskbar { ... }
X .start-menu { ... }
X .mat-icon { ... }
X .browser-* { ... }
X .control-btn { ... }
X .resize-handle { ... }

THEME CSS SHOULD CONTAIN:
+ :root { --system-*: ... }
+ :root { --system-icon-bg: ... }
+ :root { --system-icon-border: ... }
+ .system-btn { ... }
+ .system-card { ... }
+ .system-app { ... }
+ .system-desktop-taskbar { ... } (OPTIONAL - for enhanced theming)
+ .system-desktop-window { ... } (OPTIONAL - for enhanced theming)
+ .system-desktop-icon { ... } (OPTIONAL - for enhanced theming)
+ body { background: ... }
+ #themeCanvas { display: ... }

4. THEME CONFIGURATION (themes.json)
================================================================================

themes.json STRUCTURE:

{
  "your-theme-id": {
    "name": "Display Name",
    "description": "Brief description",
    "css": "themes/your-theme/styles.css",
    "script": "themes/your-theme/script.js",
    "canvas": true,
    "preview": "themes/your-theme/preview.jpg",
    "author": "Your Name",
    "version": "1.0.0",
    "settings": {
      "particles": false,
      "particleCount": 20
    }
  }
}

CONFIGURATION PROPERTIES:

Property     | Type    | Required | Description
-------------|---------|----------|------------------------------------------
name         | String  | Yes      | Human-readable theme name (shown in Settings)
description  | String  | Optional | Optional description for theme selector
css          | String  | Yes      | Path to CSS file (relative to desktop.html)
script       | String  | Optional | Path to JS file that sets window.ThemeCleanup
canvas       | Boolean | Yes      | true to show canvas, false to hide it
preview      | String  | Optional | Optional thumbnail image for theme selector
author       | String  | Optional | Theme author name
version      | String  | Optional | Theme version (semver recommended)
settings     | Object  | Optional | Optional config passed to theme scripts

PRO TIP: Access settings in your script

// In script.js - access via window.ThemeConfig
const themeId = 'your-theme-id';  // Match your themes.json key
const config = window.ThemeConfig?.[themeId]?.settings || {};
const particleCount = config.particleCount ?? 20;

NOTE: ThemeManager.themeConfig contains the full themes.json data after loading.
      Theme scripts can access it via window.ThemeConfig (set by ThemeManager).

5. CSS: COMPLETE OVERRIDE SYSTEM
================================================================================

*** CSS VARIABLES OVERRIDE BASE STYLES ***

Theme CSS overrides base styles through CSS custom properties. OS shell elements
in desktop.html reference these variables, so themes only need to override :root.

CORRECT - Variable Overrides:

:root {
  --system-bg: linear-gradient(135deg, #1a1a2e, #0f3460);
  --system-window-bg: rgba(30, 30, 46, 0.6);
  --system-accent: #7ec8e3;
  --system-icon-bg: rgba(30, 30, 46, 0.8);
  --system-icon-border: 1px solid rgba(126, 200, 80, 0.3);
}

INCORRECT - Direct Shell Styling (DO NOT DO THIS):

.icon { ... }           /* Styled in desktop.html */
.window { ... }         /* Styled in desktop.html */
.taskbar { ... }        /* Styled in desktop.html */
.start-menu { ... }     /* Styled in desktop.html */

OPTIONAL - Desktop System Class Enhancements (v2.7+):

/* These are safe because they use the .system-desktop-* namespace */
.system-desktop-taskbar {
  backdrop-filter: blur(20px);
  box-shadow: 0 0 20px rgba(0, 120, 212, 0.3);
}

.system-desktop-icon {
  backdrop-filter: blur(10px);
  transition: transform 0.2s ease;
}

.system-desktop-icon:hover {
  transform: translateY(-2px);
}

BACKGROUND IMAGE SUPPORT:

/* Use relative path from CSS file location */
html, html body, body {
  background-color: #fallback-color !important;
  background-image: url('background.png') !important;
  background-repeat: no-repeat !important;
  background-position: center center !important;
  background-attachment: fixed !important;
  background-size: cover !important;
  min-height: 100dvh !important;
  min-width: 100vw !important;
}

/* Or use absolute path from web root */
html, html body, body {
  background: #fallback-color url('/themes/your-theme/background.png') no-repeat center center fixed !important;
  background-size: cover !important;
}

ACCESSIBILITY: REDUCED MOTION

@media (prefers-reduced-motion: reduce) {
  body {
    animation: none !important;
  }
  .system-btn,
  .system-card,
  .system-app *,
  .system-desktop-icon,
  .system-desktop-window {
    transition: none !important;
    animation: none !important;
    transform: none !important;
  }
}

FALLBACK FOR NO BACKDROP-FILTER SUPPORT:

@supports not (backdrop-filter: blur(10px)) {
  .system-card,
  .system-app .card,
  .system-desktop-taskbar,
  .system-desktop-start-menu {
    background: rgba(30, 30, 46, 0.95) !important;
    backdrop-filter: none !important;
  }
}

11. LONG-PRESS INTERACTION MODEL [NEW v2.7+]
================================================================================

*** TOUCH-FIRST CONTEXT MENUS FOR DESKTOP ICONS ***

Material Icons OS v2.7+ introduces a unified long-press interaction model for
desktop icons and Start Menu items. This replaces right-click context menus
on touch devices while maintaining keyboard/mouse compatibility.

INTERACTION FLOW:
1. User presses/holds desktop icon for 500ms
2. Visual feedback: subtle scale + optional haptic vibration (navigator.vibrate)
3. Context menu appears at touch/cursor position
4. User selects action or taps outside to dismiss
5. Menu auto-cleans up with proper event listener removal

SETUPLONGPRESS() UTILITY FUNCTION:

// Location: desktop.html <script> section
// Purpose: Unified long-press detection for mouse + touch

setupLongPress(element, callbacks, options = {})

Parameters:
-------------------------------------------------------------------------------
element          | DOM Element to attach listeners to
callbacks        | Object with event handlers:
  • onLongPress  | Function(e): Called after 500ms hold (shows context menu)
  • onShortPress | Function(e): Called on quick tap/click (launches app)
  • onMove       | Function(e): Called if pointer moves during press (cancels)
  • duration     | Number: Hold duration in ms (default: 500)
  • preventContextMenu | Boolean: Suppress native contextmenu on long-press
options          | Optional config:
  • moveThreshold | Number: Pixels of movement to cancel long-press (default: 10)

Returns: cleanup() function to remove all attached listeners

Usage Example (Desktop Icon):
-------------------------------------------------------------------------------
const icon = document.createElement('div');
icon.className = 'icon system-desktop-icon';
icon.dataset.appId = 'my-app';

const launchApp = () => { /* open app logic */ };

const onLongPress = (e) => {
  e.preventDefault();
  const point = e.type.includes('touch') ? e.touches[0] : e;
  
  ContextMenu.show(
    point.clientX,
    point.clientY,
    [
      { 
        label: 'Remove from Desktop', 
        icon: 'delete_outline', 
        value: 'remove',
        danger: true 
      }
    ],
    (action) => {
      if (action === 'remove') {
        DesktopManager.removeFromDesktop('my-app');
        icon.remove();
      }
    }
  );
};

setupLongPress(icon, {
  onLongPress,
  onShortPress: launchApp,
  preventContextMenu: true
});

KEY FEATURES:
• Cross-platform: Works with mouse, touch, pen input
• Movement detection: Cancels if user drags during press
• Haptic feedback: navigator.vibrate(10) on long-press (if supported)
• Accessibility: Keyboard Enter/Space triggers short-press action
• Cleanup: Returns function to remove all event listeners

CONTEXTMENU API:

ContextMenu.show(x, y, items, onSelect, onClose)

Parameters:
-------------------------------------------------------------------------------
x, y        | Number: Screen coordinates for menu position
items       | Array: Menu item definitions:
  • label   | String: Display text
  • icon    | String: Material Icons glyph name (optional)
  • value   | Any: Identifier passed to onSelect callback
  • danger  | Boolean: Render with danger color styling (optional)
onSelect    | Function(value, item): Called when user selects menu item
onClose     | Function(): Called when menu dismisses (click outside, Escape)

Returns: Reference to created menu DOM element

Usage Example:
-------------------------------------------------------------------------------
ContextMenu.show(
  200,  // x coordinate
  150,  // y coordinate
  [
    { label: 'Open', icon: 'launch', value: 'open' },
    { label: 'Add to Desktop', icon: 'add_to_home_screen', value: 'add' },
    { label: 'Remove', icon: 'delete', value: 'remove', danger: true }

12. JAVASCRIPT & CANVAS ANIMATIONS
================================================================================

*** SCRIPT REQUIREMENTS (NON-NEGOTIABLE) ***

1. Wrap entire script in IIFE: (function() { ... })();
2. Set window.ThemeCleanup = function cleanup() {...} at global scope
3. Keep canvas transparent: use clearRect(), never fill with solid color
4. Store event handler references for proper removal in cleanup
5. Cancel all requestAnimationFrame, setInterval, setTimeout
6. Use proper canvas variable scoping (local const or this.canvas)

MINIMUM VIABLE SCRIPT TEMPLATE:

(function() {
  'use strict';
  
  const canvas = document.getElementById('themeCanvas');
  if (!canvas) { window.ThemeCleanup = function() {}; return; }
  
  const ctx = canvas.getContext('2d');
  let animationFrame = null;
  let resizeHandler = null;
  
  // HiDPI resize
  function resizeCanvas() {
    const dpr = window.devicePixelRatio || 1;
    canvas.width = window.innerWidth * dpr;
    canvas.height = window.innerHeight * dpr;
    canvas.style.width = window.innerWidth + 'px';
    canvas.style.height = window.innerHeight + 'px';
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    ctx.scale(dpr, dpr);
  }
  resizeCanvas();
  resizeHandler = resizeCanvas;
  window.addEventListener('resize', resizeHandler);
  
  // Enforce canvas layering
  canvas.style.cssText = 'position:fixed;top:0;left:0;width:100%;height:100%;z-index:0;pointer-events:none;background:transparent;';
  
  // Respect reduced motion
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
    window.ThemeCleanup = function() {};
    return;
  }
  
  // Animation loop
  function animate() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    // Draw your animation here
    animationFrame = requestAnimationFrame(animate);
    if (ThemeManager?.trackAnimation) ThemeManager.trackAnimation(animationFrame);
  }
  animate();
  
  // REQUIRED cleanup
  window.ThemeCleanup = function cleanup() {
    if (animationFrame) cancelAnimationFrame(animationFrame);
    if (resizeHandler) window.removeEventListener('resize', resizeHandler);
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    ctx.clearRect(0, 0, canvas.width, canvas.height);
  };
  
  if (ThemeManager?.registerCleanup) ThemeManager.registerCleanup(() => {});
})();

13. CLEANUP SYSTEM
================================================================================

The ThemeManager provides a robust cleanup system to prevent memory leaks 
and visual artifacts when switching themes.

HOW CLEANUP WORKS:
1. User switches theme -> ThemeManager.cleanup() is called
2. ThemeManager cancels all tracked animation frames, intervals, timeouts
3. ThemeManager executes window.ThemeCleanup function (if set by theme script)
4. ThemeManager clears canvas and removes script element
5. New theme loads -> Clean slate, no artifacts from previous theme

CLEANUP CHECKLIST FOR THEME SCRIPTS:

[ ] Cancel all requestAnimationFrame with cancelAnimationFrame()
[ ] Clear all setInterval with clearInterval()
[ ] Clear all setTimeout with clearTimeout()
[ ] Remove all event listeners with removeEventListener()
[ ] Clear canvas with ctx.clearRect()
[ ] Reset canvas transform with ctx.setTransform(1, 0, 0, 1, 0, 0)
[ ] Delete any dynamically created DOM elements
[ ] Reset any modified CSS variables to defaults

TRACKING HELPERS (Use ThemeManager API):

// In your theme script

16. COMMON PITFALLS
================================================================================

*** THEME CSS STYLING OS SHELL ELEMENTS ***
Problem: Theme CSS contains .icon, .window, .taskbar styles
Solution: Remove all OS shell selectors; use CSS variable overrides in :root

*** ICONS MISSING BACKGROUND/BORDER ***
Problem: Desktop icons have no background or border in theme
Solution: Theme must override --system-icon-bg and --system-icon-border in :root

*** BACKGROUND IMAGE NOT SHOWING ***
Problem: Background image doesn't appear
Solution: Use correct path, full background shorthand with !important, min-height:100dvh

*** CANVAS BLOCKING BACKGROUND ***
Problem: Can't see CSS background through canvas
Solution: Use clearRect() not fillRect(); set canvas CSS: background:transparent !important

*** CLEANUP NOT EXECUTING ***
Problem: Animations continue after theme switch
Solution: Ensure window.ThemeCleanup is set at global scope, not inside IIFE only

*** CANVAS REFERENCE ERROR ***
Problem: ReferenceError: canvas is not defined in cleanup
Solution: Use this.canvas in ThemeManager.cleanup(), store canvas reference properly

*** DESKTOP CLASSES NOT APPLYING ***
Problem: .system-desktop-* classes have no effect
Solution: Ensure desktop.html includes the class definitions in system-components style block

*** ICONS RENDERING AS TEXT INSTEAD OF GLYPHS ***
Problem: Icon names like "home" or "settings" display as words, not icons
Solutions:
1. Verify @font-face is declared in desktop.html <head> BEFORE .material-icons CSS
2. Check font file path: /system/fonts/MaterialIcons-Regular.ttf exists
3. Ensure .material-icons class includes font-feature-settings: 'liga'
4. Verify server serves .ttf with MIME type: font/ttf
5. Hard refresh browser (Ctrl+F5) to clear font cache
6. Check DevTools Console for font load errors (404, CORS, MIME type)
7. Confirm no Google Fonts CDN link is present (conflicts with local font)

*** LONG-PRESS NOT TRIGGERING ON TOUCH [NEW v2.7+] ***
Problem: Holding icon doesn't show context menu on mobile
Solutions:
1. Verify setupLongPress is called with touchstart/touchend listeners
2. Check preventDefault isn't blocking touch events prematurely
3. Ensure moveThreshold isn't too sensitive for finger tremor
4. Test with duration: 300 for faster feedback during development

*** CONTEXT MENU POSITIONING OFF-SCREEN [NEW v2.7+] ***
Problem: Menu appears cut off at screen edges
Solutions:
1. ContextMenu.show() auto-clamps to window.innerWidth/innerHeight
2. For custom positioning, add boundary checks:
   x = Math.min(x, window.innerWidth - menuWidth)
   y = Math.min(y, window.innerHeight - menuHeight)
3. Consider flipping menu direction based on available space

*** HAPTIC FEEDBACK TOO AGGRESSIVE [NEW v2.7+] ***
Problem: Vibration feels jarring or drains battery
Solutions:
1. Use short duration: navigator.vibrate(10) not 100ms
2. Wrap in feature detection: if (navigator.vibrate) { ... }
3. Respect prefers-reduced-motion to skip haptics entirely
4. Consider making haptics optional via theme settings

*** KEYBOARD USERS CAN'T ACCESS CONTEXT MENU [NEW v2.7+] ***
Problem: Long-press requires touch/mouse, excludes keyboard nav
Solutions:
1. Add Shift+F10 or Context Menu key handler as alternative trigger
2. Provide "Options" button in app window header as fallback
3. Document keyboard shortcuts in Help/Settings UI
4. Future: Implement focusable menu with arrow key navigation

(function() {
  'use strict';
  
  const canvas = document.getElementById('themeCanvas');
  if (!canvas) { window.ThemeCleanup = function() {}; return; }
  
  const ctx = canvas.getContext('2d');
  let animationFrame = null;
  let resizeHandler = null;
  
  // HiDPI resize
  function resizeCanvas() {
    const dpr = window.devicePixelRatio || 1;
    canvas.width = window.innerWidth * dpr;
    canvas.height = window.innerHeight * dpr;
    canvas.style.width = window.innerWidth + 'px';
    canvas.style.height = window.innerHeight + 'px';
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    ctx.scale(dpr, dpr);
  }
  resizeCanvas();
  resizeHandler = resizeCanvas;
  window.addEventListener('resize', resizeHandler);
  
  // Enforce canvas layering
  canvas.style.cssText = 'position:fixed;top:0;left:0;width:100%;height:100%;z-index:0;pointer-events:none;background:transparent;';
  
  // Respect reduced motion
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
    window.ThemeCleanup = function() {};
    return;
  }
  
  // Animation loop
  function animate() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    // Draw your animation here
    animationFrame = requestAnimationFrame(animate);
    if (ThemeManager?.trackAnimation) ThemeManager.trackAnimation(animationFrame);
  }
  animate();
  
  // REQUIRED cleanup
  window.ThemeCleanup = function cleanup() {
    if (animationFrame) cancelAnimationFrame(animationFrame);
    if (resizeHandler) window.removeEventListener('resize', resizeHandler);
    ctx.setTransform(1, 0, 0, 1, 0, 0);
    ctx.clearRect(0, 0, canvas.width, canvas.height);
  };
  
  if (ThemeManager?.registerCleanup) ThemeManager.registerCleanup(() => {});
})();

=== MATERIAL ICONS FONT SETUP (LOCAL FILE) ===

CRITICAL: DO NOT USE GOOGLE FONTS CDN

The OS is configured to use a local Material Icons font file for offline 
compatibility and performance. Using the CDN will cause icons to fail when 
offline and adds unnecessary external dependencies.

REQUIRED SETUP IN desktop.html <head>:
-------------------------------------------------------------------------------
REMOVE THIS LINE (Google Fonts CDN):
<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">

ADD THIS LOCAL @font-face DECLARATION:
<style id="material-icons-font">
@font-face {
  font-family: 'Material Icons';
  src: url('/system/fonts/MaterialIcons-Regular.ttf') format('truetype');
  font-weight: normal;
  font-style: normal;
  font-display: block;
}
</style>
-------------------------------------------------------------------------------

FILE REQUIREMENTS:
-------------------------------------------------------------------------------
Path: /system/fonts/MaterialIcons-Regular.ttf
Format: TrueType Font (.ttf)
Source: https://github.com/google/material-design-icons/blob/master/font/MaterialIcons-Regular.ttf
Size: ~180KB (uncompressed)
-------------------------------------------------------------------------------

VERIFICATION CHECKLIST:
[ ] Font file exists at /system/fonts/MaterialIcons-Regular.ttf
[ ] Server serves .ttf files with correct MIME type: font/ttf
[ ] No 404 errors in browser DevTools Network tab for the font file
[ ] Icons render correctly (text like "home" displays as glyph, not word)
[ ] Font loads without CORS errors (same-origin serving recommended)

TROUBLESHOOTING:
-------------------------------------------------------------------------------
Problem: Icons show as text ("home") instead of glyphs
Solutions:
1. Verify @font-face is declared BEFORE .material-icons CSS rules
2. Check font file path is correct (case-sensitive on Linux servers)
3. Ensure font-feature-settings: 'liga' is present in .material-icons class
4. Hard refresh browser (Ctrl+F5) to clear font cache
5. Check Console for "Failed to load resource" errors

Problem: Font fails to load (404 or CORS error)
Solutions:
1. Verify file exists at exact path /system/fonts/MaterialIcons-Regular.ttf
2. Configure server MIME type: AddType font/ttf .ttf (Apache) or 
   types { font/ttf ttf; } (Nginx)
3. For cross-origin serving, add CORS header: 
   Access-Control-Allow-Origin: *
4. Consider converting to WOFF2 for better compression + broader support

OPTIONAL: FONT FALLBACK STRATEGY
-------------------------------------------------------------------------------
For maximum reliability, add a ligature-based text fallback:

.material-icons {
  font-family: 'Material Icons', 'Material Icons Outlined', sans-serif !important;
  
  font-feature-settings: 'liga';
  -webkit-font-feature-settings: 'liga';
}

PERFORMANCE TIP: Convert TTF to WOFF2
-------------------------------------------------------------------------------
WOFF2 offers ~30% smaller file size with identical quality:

1. Convert using fonttools or online converter:
   https://github.com/google/woff2

2. Update @font-face with multiple formats for broader support:
@font-face {
  font-family: 'Material Icons';
  src: url('/system/fonts/MaterialIcons-Regular.woff2') format('woff2'),
       url('/system/fonts/MaterialIcons-Regular.ttf') format('truetype');
  font-weight: normal;
  font-style: normal;
  font-display: block;
}

3. Update themes.json or build process to reference .woff2 first
